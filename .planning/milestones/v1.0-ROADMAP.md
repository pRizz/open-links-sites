## Overview

`open-links-sites` v1.0 shipped the first operator-ready control repo around upstream `open-links`. The milestone established the canonical per-person data contract, added guided CRUD and import flows, reused upstream enrichment/build behavior through generated workspaces, and automated centralized GitHub Pages deployment plus upstream sync operations.

## Phases

## Milestone Summary

**Key Decisions**

- Keep `open-links-sites` as a thin orchestrator around upstream `open-links`.
- Use `people/<id>/` as the canonical per-person source boundary, with generated workspaces under `generated/<id>/`.
- Make the repo-local `manage-person` skill the primary operator surface, backed by deterministic Bun scripts.
- Keep deployment centralized and path-based under GitHub Pages.
- Pin upstream `open-links` state in source control and gate sync plus deploy through one shared `release:verify` contract.

**Issues Deferred**

- Richer provider-specific extraction for JavaScript-heavy link pages.
- Further optimization if the restored live Pages artifact becomes large enough to slow selective deploys materially.
- Potential upstream generalization if local asset translation boundaries become harder to maintain.

**Technical Debt Incurred**

- No critical debt was accepted for ship. Residual risks are tracked in `v1.0-MILESTONE-AUDIT.md`.
